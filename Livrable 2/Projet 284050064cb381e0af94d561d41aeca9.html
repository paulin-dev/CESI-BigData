Introduction

Présentation de l’outil Apache NiFi

Apache NiFi est un outil open-source d’orchestration de flux de données (Dataflow).

Il permet d’automatiser l’ingestion, la transformation et le chargement de données via une interface visuelle.

Sa gestion des FlowFiles (unités de données) et de la traçabilité des flux garantit la fiabilité, la reprise sur incident et la conformité RGPD.

Dans le cadre du projet CHU, NiFi joue le rôle de moteur ETL principal, orchestrant les échanges entre :

  • Sources hétérogènes : bases PostgreSQL, fichiers CSV, serveur FTP,
  • Zone de stockage cible : HDFS / Hive (entrepôt décisionnel).

Chaque job NiFi est conçu selon la logique ETL :

  • Extract : connexion et récupération des données brutes,
  • Transform : nettoyage, conversion, enrichissement et anonymisation,
  • Load : écriture vers HDFS ou Hive.

Flux d’intégration de données (Apache NiFi)

Objectif général

L’objectif de cette phase est de concevoir et automatiser les flux d’intégration des données provenant de différentes sources hétérogènes (PostgreSQL, fichiers CSV, FTP).

Ces flux sont réalisés à l’aide d’Apache NiFi, un outil d’ETL orienté flux de données permettant de gérer :

  • L’extraction des données brutes ;
  • Leur transformation (nettoyage, conversion de format, enrichissement) ;
  • Leur chargement dans le système de stockage cible (entrepôt Hive / HDFS).

Chaînage et orchestration globale

L’ensemble des jobs NiFi est orchestré dans un pipeline global :

  • Les jobs s’exécutent de manière séquentielle ou parallèle selon les dépendances (par ex. le job Job_Patient_PostgreSQL s’exécute avant Job_Satisfaction_FTP pour conserver la cohérence des identifiants).
  • Chaque job écrit les données dans une zone HDFS spécifique (/data/patients/, /data/etablissements/, etc.).
  • Les jobs sont ensuite monitorés via le Data Provenance NiFi pour garantir la traçabilité et détecter les erreurs.
  • En cas d’échec, une politique de retry automatique et d’alerte est configurée pour fiabiliser le traitement.

Conformité RGPD et sécurité

Les données manipulées étant de nature médicale, chaque flux NiFi applique les principes suivants :

  • Minimisation des données : seuls les champs utiles à l’analyse sont conservés ;
  • Anonymisation : les noms, prénoms et numéros d’identifiants patients sont hachés via SHA2 ;
  • Traçabilité : chaque FlowFile garde une provenance complète dans NiFi ;
  • Transfert sécurisé : communication entre NiFi et PostgreSQL via JDBC SSL.
